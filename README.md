项目说明文档

一、 项目概述
Audio Journey Workstation 是一个自动化新闻聚合与语音合成工作站。系统基于 FastAPI 构建后端，纯 HTML/JavaScript/CSS 构建前端。其核心功能是整合 GPT-SoVITS 文本转语音接口，配合系统内的动态背景音乐混音逻辑，将网络抓取的长文本或输入文本自动转化为复合音频文件，并提供可视化的网页控制终端供用户下发任务。

二、 核心特质
1. 前端交互界面：系统前端采用单文件原生架构编写，摒弃了常规的前端编译框架。前端实现了页面间的平滑切换、数据状态同步以及根据音频播放状态实时响应的可视化图表。
2. 自动化文本采集：系统整合了具备数据清洗功能的脚本。脚本负责并发请求 RSS 新闻源，剔除网页杂质提取正文后，将纯净文本传递至语音合成模块。
3. 动态混音算法：在语音合成结束后，系统会根据内部设定的权重比值自动匹配指定路径下的背景音乐，通过淡入淡出处理与目标语音进行重叠混流，生成最终的成品音频。
4. 解耦架构：FastAPI 作为服务端，承载了数据通信与系统级任务分发的工作。服务端支持静态资源反向代理与音频流式传输，并在后台静默调度底层的批处理脚本执行高耗时推理任务，避免阻塞通信主线程。
5. 推理控制模块：系统前端包含完整的推理参数调试平台，支持用户对超过十五项核心参数（包括温度、随机种子、切分方式、语速等）进行精确量化控制。

三、 目录结构说明
E:\pycode 目录下的核心文件职责如下：
index.html: 前端主视图文件，包含所有 DOM 骨架结构、样式定义与异步请求事务逻辑。
server.py: 后端主控程序，负责启动 API 监听端口，转发路由，扫描本地文件资产以及处理跨组件的参数落盘。
science.py 与 default copy.py: 自动化抓取、截取文本内容，并执行调用接口及音频后处理任务的 Python 脚本。
webui_submmit_request.py: 数据提交脚本，负责读取前端生成的配置文件并向远端或本地推理服务发送特定结构的数据包。
transfer_data.json: 本地环境下的数据中转文件，用于协调前后端分离状态下的非实时参数传递。
auto.bat 等批处理文件: 用于自动化部署、启动 GPT-SoVITS 大模型服务及执行异步管道任务的脚本。
otp 目录: 用于存放工作站任务执行完毕后输出的最终合成音频文件。
selected_pictures 目录: 前端界面执行动态更换皮肤或底图时所调用的绝对路径图片库。

四、 运行指南
启动准备：确认系统已安装依赖环境，如 fastapi、uvicorn、requests、newsplease 与 pydub。
启动服务：在 E:\pycode 目录下，使用命令行执行 python server.py 指令。
访问终端：当终端输出 Uvicorn 服务正在运行的提示后，打开本地浏览器，输入 127.0.0.1:8000 即可进入系统操作界面。

五、 功能模块使用方法
主控页面：负责管理音频播放与系统级任务的发起。左侧为媒体播放器与状态显示面板，右侧为系统控制命令组。点击全局或特定类别的执行按钮，系统会在后台启动自动抓取与合成流程。点击列表展示功能可显示已生成的本地历史音频文件并供用户试听。
设置页面：负责特定文本的人工干预合成控制。用户可以在透明输入框中填入目标文本，在右侧面板通过滑动条调整参数数值。信息设定完毕后提交，系统进入参数封包与合成流程。

六、 维护与开发建议
背景音乐概率调整：如需改变特定音频作为背景音乐被选中的概率，可前往对应处理脚本修改 special_rules 字典内的名称匹配及其权重浮点数。
媒体流管控：在涉及高并发的文件读写或外部网络流访问时，后端 server.py 路由中的文件代理方法可以被重写以提供更有效率的缓冲控制。
容器布局规范：如果在前端增加新的占位组件，需注意 CSS 的外边距重叠可能导致页面高度发生异常扩展，当前系统的主要隔离方案为添加边界内边距。
